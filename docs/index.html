Overview

Panchang Core is a PHP library for calculating the five limbs (Panchanga) of the Vedic calendar: Tithi (lunar day), Vara (weekday), Nakshatra (lunar mansion), Yoga (auspicious conjunction), and Karana (half tithi).

Features

  • Swiss Ephemeris via FFI for astronomical precision
  • Amanta & Purnimanta calendar systems with CalendarType enum
  • Adhika / Kshaya Maas detection (leap & deleted months)
  • 193 Festivals with classical tithi resolution and vriddhi (multi-day) handling
  • 3 Languages: English, Hindi, Gujarati — via Localization::translate()
  • Eclipse prediction: Solar & Lunar eclipses with visibility calculations
  • Muhurta evaluation: Electional astrology with transit-only mode
  • Complete Panchanga: Lagna table, Hora, Chogadiya, Rahu Kaal, Yamaganda, Gulika
  • CLI Scripts: Generate festivals, eclipses, today's panchang as JSON

System Requirements

  • PHP ≥ 8.3
  • Swiss Ephemeris shared library (libswe.so / swe.dll)
  • FFI extension enabled (ffi.enable=1 in php.ini)

Installation

Via Composer

composer require jayeshmepani/panchang-core

Laravel Integration

The package auto-discovers its service provider. Configure via config/panchang.php:

php vendor:publish --provider="JayeshMepani\\PanchangCore\\PanchangServiceProvider"

Configuration

// .env
PANCHANG_EPHE_PATH=/path/to/ephe
PANCHANG_LOCALE=en          # en, hi, gu
PANCHANG_CALENDAR_TYPE=amanta  # amanta, purnimanta

Standalone Usage

require 'vendor/autoload.php';

use JayeshMepani\PanchangCore\Traits\CliBootstrap;

CliBootstrap::init(__DIR__);
$service = CliBootstrap::makePanchangService();

Calendar Type (Amanta / Purnimanta)

The Hindu lunar calendar has two major systems for naming months:

FeatureAmanta (Amavasyant)Purnimanta
Month starts afterAmavasya (New Moon)Purnima (Full Moon)
Paksha orderShukla → KrishnaKrishna → Shukla
Used inGujarat, Maharashtra, Karnataka, TN, AP, KeralaUP, Bihar, Rajasthan, MP, Punjab
Shukla PakshaMonth NMonth N (same during Shukla)
Krishna PakshaMonth NMonth N+1 (different during Krishna)

Output Always Includes Both Month Names

Every output JSON includes both Month_Amanta and Month_Purnimanta fields, regardless of the configured Calendar_Type. The Calendar_Type field indicates which system was used for festival resolution and month-based filtering, but both month names are always available for cross-referencing.

Usage

// Via config
'calendar_type' => 'purnimanta',  // or 'amanta' (default)

// Via method parameter
use JayeshMepani\PanchangCore\Core\Enums\CalendarType;

$details = $service->getDayDetails(
    date: $today,
    lat: 23.24,
    lon: 69.66,
    tz: 'Asia/Kolkata',
    calendarType: CalendarType::Purnimanta,
);

// Via string (auto-converted)
$details = $service->getDayDetails(
    date: $today,
    lat: 23.24,
    lon: 69.66,
    tz: 'Asia/Kolkata',
    calendarType: 'purnimanta',
);

Festivals

The package includes 193 festivals with classical tithi resolution:

Festival Types

  • Tithi-based: Ekadashi, Purnima, Amavasya, Chaturthi, etc.
  • Sankranti: All 12 monthly Sankrantis (Sun enters each sign)
  • Navratri: 9-day sequences with individual deity worship
  • Weekday-based: Shravana Somvar (Monday fasting)
  • Nakshatra-based: Thai Poosam, Onam
  • Day-after: Holi (after Holika Dahan)
  • Family: BAPS Swaminarayan festival family with vriddhi handling

Adhika Maas Handling

For tithi festivals, default behavior is Nija-only unless a rule explicitly allows Adhika or marks Adhika-only. Rule metadata supports allow_adhika, allows_adhika, and adhika_only.

Vriddhi (Multi-Day) Handling

When a tithi spans multiple sunrises, the festival can appear on multiple dates. The rules_applied field in output shows which date won and why.

Eclipses

Solar and lunar eclipse prediction with visibility calculations:

  • Eclipse type: Total, Partial, Annular, Penumbral
  • Timing: Start, maximum, end (in local timezone)
  • Sutak period: Inauspicious period before eclipse
  • Visibility: Whether visible from the configured location
  • Magnitude and obscuration percentage
$eclipses = $eclipseService->getEclipsesForYear(2026, 23.24, 69.66, 'Asia/Kolkata');
// Returns array of eclipse for the year

Muhurta

Daily Muhurtas

  • Abhijit Muhurta: Most auspicious ~48 min around solar noon
  • Brahma Muhurta: Pre-dawn spiritual period
  • Vijaya Muhurta: Afternoon auspicious period
  • Godhuli Muhurta: Twilight period
  • Dur Muhurta: Inauspicious periods to avoid
  • Nishita Muhurta: Midnight spiritual period

Kaal Vela

  • Rahu Kaal: Inauspicious period ruled by Rahu
  • Yamaganda: Inauspicious period ruled by Yamaganda
  • Gulika: Inauspicious period ruled by Gulika

Electional Evaluation

getDailyMuhurtaEvaluation() provides a comprehensive auspiciousness assessment for the current day, including:

  • Overall rating (Excellent / Good / Mixed / Challenging)
  • Transit Panchanga quality score
  • Combustion check for all planets
  • Auspicious / Inauspicious windows
  • Special observations (eclipses, Sankranti, etc.)
