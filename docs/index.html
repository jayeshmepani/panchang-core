Panchang Core

Introduction

Authentic Vedic Panchanga calculation engine with Swiss Ephemeris precision

Overview

Panchang Core is a comprehensive PHP package for calculating Vedic Panchanga elements, Muhūrtas, and Hindu festivals with maximum astronomical precision. Built on Swiss Ephemeris FFI, it provides zero-tolerance calculations for authentic Vedic astrology applications.

🔮 Complete Panchanga

Tithi, Vara, Nakṣatra, Yoga, Karaṇa with precise fractions

📅 50+ Festivals

Holikā Dahan, Rāma Navamī, Kṛṣṇa Janmāṣṭamī, Dīpāvalī with tradition profiles

⏰ Muhūrta

Abhijit, Brahma Muhūrta, Rahu Kāla, Gulika, Yamaganda

🌍 Regional Support

Smarta, Vaishnava, North, South, Bengal, Maharashtra, Tamil traditions

Installation

Composer Install 
composer require jayeshmepani/panchang-core

Requirements

  • PHP 8.3+ (uses typed constants, readonly classes, enums)
  • Swiss Ephemeris FFI (jayeshmepani/swiss-ephemeris-ffi)
  • Carbon (nesbot/carbon)
  • FFI Extension (for Swiss Ephemeris)

Quick Start

Standalone Usage 
<?php
require 'vendor/autoload.php';

use JayeshMepani\PanchangCore\Panchanga\PanchangService;
use JayeshMepani\PanchangCore\Astronomy\AstronomyService;
use JayeshMepani\PanchangCore\Astronomy\SunService;
use JayeshMepani\PanchangCore\Panchanga\PanchangaEngine;
use JayeshMepani\PanchangCore\Panchanga\MuhurtaService;
use SwissEph\FFI\SwissEphFFI;
use Carbon\CarbonImmutable;

// Initialize services
$sweph = new SwissEphFFI();
$panchang = new PanchangService(
    $sweph,
    new SunService($sweph),
    new AstronomyService($sweph),
    new PanchangaEngine(),
    new MuhurtaService()
);

// Calculate panchanga
$date = CarbonImmutable::parse('2026-03-24');
$details = $panchang->getDayDetails(
    date: $date,
    lat: 23.2472446,
    lon: 69.668339,
    tz: 'Asia/Kolkata'
);

echo "Tithi: " . $details['Tithi']['name'] . "\n";
echo "Nakshatra: " . $details['Nakshatra']['name'] . "\n";

Configuration

Global settings and formatting preferences available in config/panchang.php.

Overview

When running inside Laravel, you can publish the configuration file to config/panchang.php. These settings control the default formatting across the entire package output, allowing you to globally enforce timezone notations, unit systems, and double precision decimals.

Default Calculation Options

The defaults array configures formatting representations. The acceptable values are:

Setting Description Acceptable Values
measurement_system The unit system used for generic dimensions and lengths (e.g., elevation, distances). indian_metric: Metric system (meters, kilometers)
Example: Elevation: "150 meters"

western: Imperial system (feet, miles)
Example: Elevation: "492 feet"
date_time_format Controls the string formatting of generic date-time outputs. indian_12h: "24/03/2026 06:14:00 AM"
indian_24h: "24/03/2026 06:14:00"
iso8601: "2026-03-24T06:14:00+05:30"
time_notation Global time output behavior (e.g. for sunrise, sunset, and muhurtas). 12h: "06:14:00 AM", "05:32:00 PM"
24h: "06:14:00", "17:32:00"
coordinate_format Format for geographical coordinates (latitude and longitude). decimal: 23.2472446, 69.668339
dms: "23°14'50.1\"N, 69°40'06.0\"E"
angle_unit Format for astronomical degrees (e.g. planet longitudes, Ascendant, Ayanamsa). degree: 350.5234°
dms: "350°31'24\""
duration_format How lengths of time are formatted in output payloads. mixed: A human-readable string combining units (e.g. "1h 30m 0s")
minutes: Total duration as decimal minutes (e.g. 90.0)
seconds: Total duration as decimal seconds (e.g. 5400.0)
hours: Total duration as decimal hours (e.g. 1.5)
number_precision The number of decimal points to maintain for all floating-point and angular outputs. Maximum accuracy is guaranteed internally via IEEE 754 precision. Integer (Default is 16 for maximum double precision).

Festival Calculation Settings

The festivals array provides defaults for region and tradition if they are not explicitly supplied during calculation.

Setting Description Acceptable Values
default_tradition The default sect or tradition used to resolve conflicting festival rules (e.g. Janmashtami timing). Smarta, Vaishnava
default_region The default geographical region for festival rule variations. North, South, Bengal, Maharashtra, Tamil

Nakshatra

Enum: JayeshMepani\PanchangCore\Core\Enums\Nakshatra

Represents the 27 lunar mansions in Vedic astrology. Each nakṣatra spans 13°20' of the zodiac.

📋 Quick Usage

use JayeshMepani\PanchangCore\Core\Enums\Nakshatra;

// Get nakshatra from moon longitude
$nakshatra = Nakshatra::fromLongitude(125.3); // Magha
echo $nakshatra->getName(); // "Magha"
echo $nakshatra->getDeity(); // "Pitṛs"
echo $nakshatra->getRulingPlanet(); // "Sun"
echo $nakshatra->getSymbol(); // "Royal Throne"

// Get pada (quarter)
$pada = Nakshatra::getPada(125.3); // 2
echo "Pada: $pada\n"; // Pada: 2

// Get longitude range
$range = Nakshatra::Ashwini->getLongitudeRange();
echo $range['start'] . "° - " . $range['end'] . "°\n"; // 0° - 13.333°

Tithi

Enum: JayeshMepani\PanchangCore\Core\Enums\Tithi

Represents the 30 lunar days in a Hindu lunar month. Each tithi is completed when the Moon gains 12° on the Sun.

📋 Quick Usage

use JayeshMepani\PanchangCore\Core\Enums\Tithi;

// Get tithi by index (1-30)
$tithi = Tithi::from(12); // Shukla Dwadashi
echo $tithi->getName(); // "Dwadashi"
echo $tithi->getPaksha()->name; // "Shukla"

// Check paksha
if ($tithi->getPaksha() === Paksha::Shukla) {
    echo "Waxing moon phase\n";
}

// Get normalized index (1-15 for both pakshas)
echo $tithi->getNormalizedIndex(); // 12

Get all festivals for a date with tradition/region support

public function getFestivals(CarbonImmutable $date, float $lat, float $lon, string $tz, string $tradition = 'Smarta', string $region = 'North', float $elevation = 0.0): array
Parameters:
  • $date (CarbonImmutable) - Date to calculate
  • $lat (float) - Latitude
  • $lon (float) - Longitude
  • $tz (string) - Timezone
  • $tradition (string) - 'Smarta' or 'Vaishnava' (default: 'Smarta')
  • $region (string) - 'North', 'South', 'Bengal', 'Maharashtra', 'Tamil' (default: 'North')
  • $elevation (float) - Elevation in meters
Returns: array - Array of festival names and details
Example: 
$festivals = $festivalService->getFestivals(
    date: CarbonImmutable::parse('2026-03-24'),
    lat: 23.2472446,
    lon: 69.668339,
    tz: 'Asia/Kolkata',
    tradition: 'Smarta',
    region: 'North'
);

foreach ($festivals as $festival) {
    echo $festival['name'] . "\n";
}

use JayeshMepani\PanchangCore\Facades\Panchang;

// Get complete panchanga

// Access data

Service Container

Or inject via constructor:

use JayeshMepani\PanchangCore\Panchanga\PanchangService;

class PanchangController extends Controller
{

        private PanchangService $panchang

}

Best Practices

try {
    $details = Panchang::getDayDetails(
        date: $date,
        lat: $lat, // Validate: -90 to 90
        lon: $lon, // Validate: -180 to 180
        tz: $tz // Validate: timezone_exists($tz)
    );
} catch (RuntimeException $e) {
    // Log error and show user-friendly message
    Log::error('Panchang calculation failed', [
        'error' => $e->getMessage(),
        'location' => compact('lat', 'lon', 'tz')
    ]);
    
    return response()->json([
        'error' => 'Unable to calculate panchang for this location'
    ], 500);
}

FAQ & Troubleshooting

Common questions and solutions

Q: Why are my calculations different from other panchang apps?

A: Differences can occur due to:

  • Ayanamsa: We use Lahiri by default. Change via config if needed.
  • Location: Ensure exact latitude/longitude
  • Tradition: Smarta vs Vaishnava have different rules
  • Tithi calculation: We use Swiss Ephemeris for maximum precision

Q: How do I change the ayanamsa?

A: Set in config or pass to getDayDetails():

// In config/panchang.php
'ayanamsa' => 'RAMAN', // Options: LAHIRI, RAMAN, KRISHNAMURTI

// Or per-call
$details = Panchang::getDayDetails(
    date: $date,
    lat: $lat,
    lon: $lon,
    tz: $tz,
    ayanamsaAt: CarbonImmutable::now() // Custom calculation time
);

Q: Why is Janmashtami on different dates?

A: This is normal! Smarta tradition celebrates on one day (based on ashtami at sunrise), Vaishnava on another (based on ashtami + Rohini nakshatra at midnight). Use the tradition parameter to get the correct date for your tradition.

Q: How accurate are the calculations?

A: We use Swiss Ephemeris which provides accuracy to within 1 arcsecond for planetary positions. All calculations use IEEE 754 double precision (53-bit significand) with no intermediate rounding.

Q: Can I use this for historical dates?

A: Yes! Swiss Ephemeris supports dates from 3000 BCE to 3000 CE. Just pass any valid CarbonImmutable date.
